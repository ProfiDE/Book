# Contributing to ProfiDE Book

Thank you for helping improve this book.

This repository accepts content updates, corrections, and new chapters through an issue-first workflow so changes stay focused and easy to review.

## Before you start

1. Open or find an issue that describes the change you want to make.
2. Keep your work limited to the scope of that issue.
3. Reference the issue in your pull request.

If you want to work on several unrelated additions, open a separate issue for each topic.

## Repository layout

- `Chapters.md` is the main index for the book.
- Top-level folders such as `Security/` and `Story/` are book sections.
- Markdown files inside those folders are individual chapters.

## Adding or updating content

When you contribute, follow these expectations:

- Write in Markdown.
- Use clear titles and plain language.
- Keep the structure of existing sections and files consistent.
- Avoid broad reorganizations unless the issue explicitly asks for them.

### Adding a new chapter

1. Place the new Markdown file in the appropriate top-level section folder.
2. If the section does not exist yet, add a new top-level folder with a descriptive name.
3. Update `Chapters.md` so readers can find the new page.
4. Add a short summary for the chapter in the index table.

### Editing an existing chapter

- Preserve the original topic and intent unless the issue says otherwise.
- Prefer focused corrections, clarifications, or expansions.
- Check links, headings, and formatting before you submit.

## Pull requests

Before opening a pull request:

- Make sure the change matches the linked issue.
- Update `Chapters.md` if you added, moved, or renamed a chapter.
- Review your Markdown for formatting mistakes and broken links.

In the pull request description, include:

- A short summary of what changed
- The related issue number
- Any notes reviewers should know about structure or section placement

## Community notes

- Be respectful and constructive in issues and pull requests.
- Do not add content that is unsafe, unlawful, or misleading.
- If you are unsure where a chapter belongs, open an issue first and ask there.
